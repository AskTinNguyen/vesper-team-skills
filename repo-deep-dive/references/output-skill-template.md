# Output Template: Generated SKILL.md

Use this template when generating the `SKILL.md` for a repo-knowledge package. Replace all `{placeholders}` with actual findings from the analysis. Remove sections that don't apply.

The generated SKILL.md must:
- Have valid YAML frontmatter with `name` and `description` (third-person)
- Use XML body tags
- Stay under 500 lines
- Use imperative/infinitive writing style throughout
- Route to reference files rather than containing all knowledge inline

## Template Notes

### Keeping Under 500 Lines

The generated SKILL.md should be a **router**, not an encyclopedia. Follow these guidelines:

- `<essential_principles>`: 3-5 key principles only, not exhaustive
- `<quick_reference>`: Top 3 patterns, most common conventions only
- `<routing>`: Table pointing to reference files
- Reference files hold the detailed knowledge

### Writing Style Checklist

- [ ] All instructions use imperative/infinitive form ("Apply X" not "You should apply X")
- [ ] Description uses third person ("This skill should be used when...")
- [ ] No second person ("you") in SKILL.md body
- [ ] Objective, instructional language throughout

### Filling the Template

1. Start with `<essential_principles>` — identify what makes this codebase unique
2. Build `<quick_reference>` from the most frequently referenced conventions
3. Verify `<routing>` table maps to actual reference files
4. Write `<success_criteria>` as testable assertions
5. Count lines — trim if over 500
