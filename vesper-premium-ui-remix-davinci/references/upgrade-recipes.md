# Upgrade Recipes

Use these recipes after selecting references.

## Settings / Preferences
### Good source references
- `settings-page`
- `add-product-form`
- `notifications`
- `roles-permissions`

### Upgrade moves
- simplify side navigation labels and section count
- convert dense fields into clearer section groups
- strengthen title + description hierarchy
- reduce equal-weight switches and buttons
- add one stable save flow per section or one global save area
- improve mobile behavior by collapsing secondary navigation cleanly

### Vesper result
A settings surface that feels calm, credible, and easy to scan — less like an admin demo, more like a premium workspace.

## Chat / AI Workspace
### Good source references
- `ai-chat`
- `chat-app`
- `chat-ui-widgets`
- `mail-app`

### Upgrade moves
- simplify chrome around the composer and thread
- make the composer the clear primary action zone
- move advanced controls into drawers, popovers, or secondary bars
- strengthen output containers and source/tool grouping
- avoid “futuristic” styling shortcuts

### Vesper result
An AI workspace that feels oriented, trustworthy, and fluid, with obvious next actions and minimal intimidation.

## Dashboard / Work Surface
### Good source references
- `admin-dashboard`
- `tasks`
- `roadmap`
- `search-results`

### Upgrade moves
- reduce decorative metrics if they distract from the primary workflow
- enforce stronger grouping and container logic
- use one hero area and quieter secondary cards
- normalize badge/status usage
- avoid over-dense control clusters

### Vesper result
A dashboard that feels editorial and guided rather than like a wall of widgets.

## Forms / Creation Flows
### Good source references
- `add-product-form`
- `multi-step-form`
- `survey-form`
- `checkout-page`

### Upgrade moves
- convert arbitrary card breaks into meaningful sections
- tighten helper text and validation copy
- make one clear completion action
- reserve richer surfaces for important groupings only
- reduce side-rail noise unless it genuinely helps completion

### Vesper result
A form flow that feels supportive and polished, with less cognitive load and stronger completion momentum.

## Hero / Landing / Empty State
### Good source references
- `saas-landing-page`
- `hero-sections`
- `coming-soon-page`
- `empty-states`

### Upgrade moves
- remove generic startup marketing tropes
- make the headline do real conceptual work
- use stronger typography contrast and warmer surfaces
- keep CTA count low
- support the hero with one or two meaningful proof points instead of many decorative blocks

### Vesper result
A warm, restrained landing or empty state that feels premium and confident rather than templated.

## Profile / Detail / Knowledge Surface
### Good source references
- `profile-page`
- `blog-detail-page`
- `directory-list`
- `help-center`

### Upgrade moves
- elevate identity or core content first
- reduce ornamental metadata treatment
- improve line length and sectional rhythm
- use containers only where they clarify meaning
- make supporting actions visible but quieter than content

### Vesper result
A readable, composed detail surface with strong hierarchy and low visual friction.

## Recipe Rule
For any recipe, keep this order:
1. choose the right structure
2. simplify the action model
3. translate to Vesper surfaces and typography
4. polish interaction states
5. verify light/dark accessibility and emotional fit
