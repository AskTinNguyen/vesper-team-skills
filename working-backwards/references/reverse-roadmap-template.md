# Reverse Roadmap Template

Start at launch success, then step backward until the next immediate experiment is obvious.

## Mode Guidance
- `Lite` and `Standard`: keep this to 3 stages unless an extra gate materially changes the decision.
- `Deep`: add extra stages only when additional gating matters.

## Reverse Roadmap

| Stage | Decision to Unlock Next Step | Evidence Required | Fastest Experiment | Owner | Target Date |
|-------|------------------------------|-------------------|--------------------|-------|-------------|
| Launch readiness | | | | | |
| Proof point before default or broader rollout | | | | | |
| Immediate next experiment | | | | | |

## Checklist (Pass All)

- Every major assumption is mapped to evidence.
- At least one fast experiment exists for each major unproven assumption.
- Irreversible commitments are delayed until evidence is collected.
- An owner and target date are listed for each stage.
- If you add more than 3 stages, the extra stage must change a real decision gate.
