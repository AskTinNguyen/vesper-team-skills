# Related UI/UX Skills

Load only the section you need. This file is a routing map, not a replacement for the underlying skills.

## `polish`

Use when the feature is already functionally complete and the remaining issues are:

- spacing rhythm
- alignment
- readability
- copy consistency
- interaction-state completeness
- final fit and finish

Best used near the end of the loop.

## `distill`

Use when the screenshot shows too much:

- redundant structure
- clutter
- too many controls
- too many containers
- too much visual competition

Best used when simplification matters more than decoration.

## `delight`

Use when the UI works but feels emotionally flat:

- empty states feel generic
- success moments feel dead
- the page lacks personality
- there is room for tasteful surprise

Best used after the structure is already sound.

## `harden`

Use when the visual review hints at product reality problems:

- long text overflow
- cramped translations
- broken empty/error states
- brittle loading states
- poor resilience under edge cases

Best used before final polish if robustness is in question.

## `optimize`

Use when the experience feels heavy or sluggish:

- slow route load
- slow interaction feedback
- janky animation
- oversized assets
- expensive rendering

Best used when screenshot polish alone cannot solve the user experience problem.

## `quieter`

Use when the UI is too loud:

- oversized type
- too much contrast
- too much saturation
- too much visual weight
- aggressive decoration or motion

Best used when the experience needs refinement through restraint.

## `audit`

Use when you need a diagnosis before making changes:

- accessibility uncertainty
- theming inconsistency
- responsive risk
- performance suspicion
- general “what is wrong here?” situations

Best used at the start of a substantial UI rework.

## `animate`

Use when motion is the missing layer:

- state changes feel abrupt
- interactions lack feedback
- navigation feels static
- the feature needs meaningful micro-interactions

Best used after layout and hierarchy are in place.

## Default Selection Rules

If the right companion skill is not obvious:

1. Start with `audit`.
2. If the UI is crowded, use `distill`.
3. If it is loud, use `quieter`.
4. If it is fragile, use `harden`.
5. If it is slow, use `optimize`.
6. If it is static, use `animate`.
7. If it is emotionally flat, use `delight`.
8. Finish with `polish`.
