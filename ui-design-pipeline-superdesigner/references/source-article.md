## Key Claims to Preserve
1. The bottleneck is iteration speed between idea, mockup, and implementation.
2. Manual work is best for micro-adjustments; AI/CLI is best for system-level generation.
3. A fixed sequence (frame → baseline → variants → pick → implement → track) improves quality and repeatability.
4. Generated output should be treated as shippable artifacts, not just visual concepts.

## Thought-Provoking Excerpts
- "My rule: manual for the micro, CLI for the system"

- "The point isn't to pick one based on gut feeling, but to choose the one that best serves the use case."
- "It generates artifacts you can commit, modify, and ship."
