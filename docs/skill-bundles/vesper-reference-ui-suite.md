# Vesper Reference UI Suite

A **3-skill workflow** for turning reference examples into premium Vesper UI.

## Skills in the suite

### 1. `shadcn-component-reference-davinci`
**Purpose:** Find and extract the strongest UI reference examples.

Use it to:
- choose 2–3 relevant source examples
- identify reusable layout, interaction, and composition patterns
- avoid literal copying

### 2. `vesper-premium-ui-remix-davinci`
**Purpose:** Translate borrowed patterns into **premium Vesper-style UI**.

Use it to:
- reinterpret references through Vesper's warm editorial design language
- upgrade hierarchy, spacing, copy, contrast, and tactility
- remove demo-grade polish and generic AI/SaaS aesthetics

### 3. `vesper-reference-ui-pipeline-davinci`
**Purpose:** Orchestrate the sequence across reference selection, Vesper translation, implementation, and polish.

Use it to:
- decide the correct order of work
- prevent jumping from references straight to code
- keep discovery, translation, implementation, and polish clearly separated

## Recommended order
1. **Harvest references** with `shadcn-component-reference-davinci`
2. **Translate to Vesper** with `vesper-premium-ui-remix-davinci`
3. **Run the full sequence** with `vesper-reference-ui-pipeline-davinci`
4. **Implement** with `gestalt-frontend-design` and `ui-design-brain`
5. **Finish** with `polish` and optional refinement skills (`clarify`, `harden`, `animate`, `adapt`, `optimize`)

## When to use this suite
Use the suite when:
- starting from external inspiration or example repos
- adapting shadcn-style references into a real product surface
- building UI that should feel distinctly **Vesper** rather than demo-derived
- the work spans sourcing, redesign, implementation, and final polish

## Anti-patterns this suite prevents
- copying source examples literally
- polishing before hierarchy and grouping are solved
- blaming implementation quality for weak source references
- ending with UI that still feels like a dressed-up demo

## Final test
A successful run through this suite should produce UI that is:
- **clearer** than the source example
- **warmer** than the source example
- **more product-specific** than the source example
- **more premium** than the source example
- unmistakably **Vesper**
